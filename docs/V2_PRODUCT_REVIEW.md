# Retra 2.0 Product Review

Date: 2026-07-22

## Product decision

Retra 2.0 is not a ROM storefront. It is a local-first emulator and archive manager for games the player is entitled to use, plus directly redistributable homebrew. This is a product-quality boundary, not a cosmetic disclaimer: commercial Pokémon ROMs are not bundled, indexed, mirrored, or downloaded by the app.

The useful FireRed flow is now:

1. Import a lawful personal `.gba` backup.
2. Validate the GBA header and copy the game into content-addressed private storage.
3. Record SHA-256, SHA-1, CRC-32, size, game code, and revision.
4. Optionally sync Libretro metadata and identify the exact dump.
5. Install a matching Libretro `.cht` file, convert it to Retra's restricted format, and bind it to the exact ROM SHA-256.
6. Preserve a protected state before cheat activation.

HeartGold, SoulSilver, and Platinum are Nintendo DS titles. Retra 2.0 remains a GBA emulator and rejects `.nds` files with a clear explanation rather than pretending to support them.

## What is real in 2.0

- Homebrew Hub search, pagination, source pages, and eligible one-tap GBA installs.
- Exact local ROM identity using SHA-256, SHA-1, CRC-32, file size, game code, and revision.
- Libretro No-Intro DAT synchronization and exact metadata matching.
- Libretro/RetroArch `.cht` import with strict parsing, unsupported placeholder filtering, conversion, and ROM binding.
- Serialized ROM imports, unique SHA-256 storage, atomic writes, duplicate handling, and failed-database-write cleanup.
- Room schema 6 and non-destructive migration from schema 5.
- A four-stage onboarding flow that explains ownership, provenance, local identity, and what Retra will not do.
- Separate Profile and Settings experiences. Settings includes an About page naming **Prashant Chataut** as developer.
- Release signing secrets removed from the repository. Production signing must come from private CI or a store workflow.

## Critical gaps before calling Retra production-ready

1. An Android build has not been compiled in this sandbox because Gradle could not be downloaded. Compose/Hilt/Room compilation still needs CI confirmation.
2. Reviewed mGBA binaries must be packaged for every supported ABI and exercised on real devices.
3. Room schema 6 export and migration instrumentation tests must be generated by a successful Android build.
4. Homebrew Hub currently supplies attached binaries and license metadata but not an expected SHA-256 in its documented file object. Retra therefore validates the GBA structure and records the downloaded SHA-256; it must not describe this source as checksum-pinned.
5. Community cheats are data, not guarantees. A correct title match can still contain unsafe or inaccurate codes, so warnings, reversibility, and per-ROM binding remain mandatory.
6. Google identity is device-local scaffolding until a backend validates tokens and nonces. It must not unlock trusted cloud or social privileges yet.

## UX assessment

The current Archive Glass direction is materially better than a collection of independent glossy cards because navigation, hierarchy, source provenance, and player safety now share one system. The main remaining UX risk is density: Discover and game details expose many advanced actions. The next pass should use progressive disclosure, keeping the first screen focused on Play, Resume, Save health, and one recommended action.
