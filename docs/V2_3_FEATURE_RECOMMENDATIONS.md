# Retra 2.3 Feature Recommendations

## Highest priority

### 1. Controller Studio

Persist mappings by controller descriptor and optionally override them per game. Include live input visualization, analog calibration, trigger thresholds, conflict detection, one-handed presets, and exportable profiles.

### 2. Save Timeline

Turn saves into an understandable history: named snapshots, automatic milestones, screenshot thumbnails, core version, game hash, timestamp, cheat state, retention policy, and one-tap preview/rollback. Never overwrite the only known-good state.

### 3. Measured performance advisor

Collect frame-time percentiles, audio underruns, rewind pressure, thermal state, and battery impact locally. Suggest a profile only after a meaningful sample. Expose the evidence instead of a decorative “boost” switch.

### 4. Game-specific achievements

Integrate rcheevos only after core memory maps and game identity are exact. Separate local Retra goals from network achievements. Support offline queues, token privacy, hardcore restrictions, and clear unsupported-game states.

### 5. Compatibility notebook

Let users record per-game notes such as “stable,” “audio issue,” or “requires interpreter mode.” Optional uploads should contain app/core version, device class, settings, and a recognized game ID—not ROM bytes, save data, paths, or account identifiers.

## Strong secondary features

- A visible patch-lineage graph and reproducible patch recipe export.
- Save-at-risk shelf linked directly to restore actions.
- Per-game launch profiles for scaling, controls, shader-free display settings, audio latency, and speed.
- Search across title, canonical title, game code, creator, tags, collections, patch name, and provenance.
- Tablet/foldable player layouts after phone controls pass usability testing.
- GB/GBC as separate pinned cores and save namespaces.

## Features to postpone

- Nintendo DS until a complete dual-screen/touch architecture exists.
- Netplay until deterministic synchronization and recovery are proven.
- Social feeds, AI labels, or cosmetic currency before saves, controls, and core stability are excellent.
