# Native/mobile delivery rules

- Use vector icons, one icon family, consistent icon sizes and visual weight.
- Android interactive targets are at least 48×48dp with at least 8dp separation.
- Provide visible pressed, focused, disabled, loading, success, and error states.
- Use semantic theme tokens; avoid per-screen raw colors.
- Maintain WCAG-readable contrast in light, dark, and OLED themes.
- Respect status/navigation bars, cutouts, gesture insets, orientation, and adaptive gutters.
- Use a 4/8dp rhythm and readable text measure on tablets.
- Screen-reader order must match visual order; icon-only controls need labels.
- Motion is causal, interruptible, normally 150–300ms, and disabled/reduced when requested.
- Validate small phone, large phone, landscape, tablet, large text, reduced motion, and both theme families.
